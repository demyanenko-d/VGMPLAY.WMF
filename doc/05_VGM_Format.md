# VGM Format Reference для OPL3

## Обзор формата VGM

VGM (Video Game Music) - это формат для хранения логов звуковых чипов из игровых консолей и компьютеров.

### Основные характеристики

- Расширение файла: **.vgm** или **.vgz** (сжатый gzip)
- Содержит команды записи в регистры звуковых чипов
- Поддерживает множество чипов, включая YM2612, YM2413, YM3812, YMF262 (OPL3)
- Точность воспроизведения оригинального звука

## Структура заголовка VGM

### Заголовок файла (256 байт)

| Offset | Размер | Название | Описание |
|--------|--------|----------|----------|
| #00 | 4 | Ident | "Vgm " (0x56 0x67 0x6d 0x20) |
| #04 | 4 | EoF offset | Смещение до конца файла (относительно этой позиции) |
| #08 | 4 | Version | Версия формата (например, 0x00000150 для v1.50) |
| #0C | 4 | SN76489 clock | Частота SN76489 (0 если не используется) |
| #10 | 4 | YM2413 clock | Частота YM2413 (0 если не используется) |
| #14 | 4 | GD3 offset | Смещение до тега GD3 |
| #18 | 4 | Total samples | Общее количество семплов |
| #1C | 4 | Loop offset | Смещение до точки зацикливания |
| #20 | 4 | Loop samples | Количество семплов в цикле |
| #24 | 4 | Rate | Частота обновления (обычно 50 или 60 Гц) |
| #34 | 4 | YM2612 clock | Частота YM2612 |
| #4C | 4 | YM2151 clock | Частота YM2151 |
| #50 | 4 | VGM data offset | Смещение до VGM данных (v1.50+) |
| #5C | 4 | YM3812 clock | **Частота YM3812 (OPL2)** |
| #60 | 4 | YM3526 clock | Частота YM3526 (OPL) |
| #64 | 4 | Y8950 clock | Частота Y8950 (MSX-Audio) |
| #68 | 4 | YMF262 clock | **Частота YMF262 (OPL3)** ← ВАЖНО! |
| #6C | 4 | YMF278B clock | Частота YMF278B (OPL4) |

**Примечание:** Для OPL3 достаточно проверить offset #68 (YMF262 clock). Если он не равен 0, файл содержит данные для OPL3.

### Смещения данных

Начиная с версии 1.50:
- Если VGM data offset (#50) = 0x00, данные начинаются с #40
- Если VGM data offset != 0, данные начинаются с (#50 + #34)

Для старых версий (< 1.50):
- Данные всегда начинаются с #40

## VGM команды для OPL3

### Базовая структура команд

Каждая команда начинается с байта-идентификатора, за которым следуют параметры.

### Команды записи в OPL3

| Команд | Размер | Параметры | Описание |
|--------|--------|-----------|----------|
| **0x5E** | 3 | aa dd | Запись в YMF262 (OPL3) порт 0 |
| **0x5F** | 3 | aa dd | Запись в YMF262 (OPL3) порт 1 |

Где:
- **aa** - адрес регистра
- **dd** - значение данных

### Команды задержки

| Команд | Размер | Параметры | Описание |
|--------|--------|-----------|----------|
| **0x61** | 3 | nn nn | Задержка на nn семплов (16-bit little-endian) |
| **0x62** | 1 | - | Задержка на 735 семплов (1/60 сек @ 44.1kHz) |
| **0x63** | 1 | - | Задержка на 882 семпла (1/50 сек @ 44.1kHz) |
| **0x70-0x7F** | 1 | - | Задержка на (n-0x6F) семплов (1-16) |

### Команды управления

| Команд | Описание |
|--------|----------|
| **0x66** | Конец данных VGM |
| **0x67** | Блок данных (PCM и др.) |

## Частота воспроизведения

VGM использует частоту семплирования 44100 Гц.

Задержки указываются в семплах:
- **1 секунда** = 44100 семплов
- **1/50 секунды** (PAL) = 882 семпла
- **1/60 секунды** (NTSC) = 735 семплов

Для ZX Spectrum (50 Гц):
```
Семплов на фрейм = 44100 / 50 = 882
```

## Пример разбора VGM для OPL3

```asm
; Проверка заголовка VGM
CHECK_VGM:
    LD HL,FILE_BUFFER
    
    ; Проверка идентификатора "Vgm "
    LD A,(HL)
    CP 'V'
    RET NZ
    INC HL
    LD A,(HL)
    CP 'g'
    RET NZ
    INC HL
    LD A,(HL)
    CP 'm'
    RET NZ
    INC HL
    LD A,(HL)
    CP ' '
    RET NZ
    
    ; Проверка наличия OPL3
    LD HL,FILE_BUFFER+#68  ; YMF262 clock offset
    LD A,(HL)
    INC HL
    OR (HL)
    INC HL
    OR (HL)
    INC HL
    OR (HL)
    RET Z                  ; Нет OPL3 данных
    
    ; Файл содержит OPL3!
    ; Получить смещение данных
    LD HL,FILE_BUFFER+#50  ; VGM data offset
    LD E,(HL)
    INC HL
    LD D,(HL)
    INC HL
    LD C,(HL)
    INC HL
    LD B,(HL)
    
    ; Проверить версию
    LD A,B
    OR C
    JR Z,OLD_VERSION
    
    ; v1.50+ : данные = offset + #34
    LD HL,#34
    ADD HL,DE
    ; BC:HL = смещение данных
    RET

OLD_VERSION:
    ; Старая версия: данные с #40
    LD HL,#40
    LD BC,0
    RET

; Воспроизведение VGM
PLAY_VGM:
    LD HL,(VGM_DATA_PTR)  ; Указатель на данные

PLAY_LOOP:
    LD A,(HL)             ; Получить команду
    INC HL
    
    CP #66                ; Конец файла?
    RET Z
    
    CP #5E                ; OPL3 порт 0?
    JR Z,OPL3_PORT0
    
    CP #5F                ; OPL3 порт 1?
    JR Z,OPL3_PORT1
    
    CP #61                ; Задержка 16-bit?
    JR Z,DELAY_16BIT
    
    CP #62                ; Задержка 1/60?
    JR Z,DELAY_60HZ
    
    CP #63                ; Задержка 1/50?
    JR Z,DELAY_50HZ
    
    CP #70                ; Короткая задержка?
    JR C,PLAY_LOOP
    CP #80
    JR NC,PLAY_LOOP
    
    ; Задержка 1-16 семплов
    SUB #6F
    LD (DELAY_COUNTER),A
    JR PLAY_LOOP

OPL3_PORT0:
    ; Запись в порт 0 (C4/C5)
    LD A,(HL)             ; Регистр
    INC HL
    LD C,A
    LD A,(HL)             ; Данные
    INC HL
    PUSH HL
    LD B,#C4              ; Порт адреса
    OUT (C),C             ; Записать адрес
    LD B,#C5              ; Порт данных
    OUT (C),A             ; Записать данные
    POP HL
    JR PLAY_LOOP

OPL3_PORT1:
    ; Запись в порт 1 (C6/C7)
    LD A,(HL)             ; Регистр
    INC HL
    LD C,A
    LD A,(HL)             ; Данные
    INC HL
    PUSH HL
    LD B,#C6              ; Порт адреса
    OUT (C),C             ; Записать адрес
    LD B,#C7              ; Порт данных
    OUT (C),A             ; Записать данные
    POP HL
    JR PLAY_LOOP

DELAY_16BIT:
    ; Задержка на nn семплов
    LD E,(HL)
    INC HL
    LD D,(HL)
    INC HL
    LD (DELAY_COUNTER),DE
    ; Обработать задержку...
    JR PLAY_LOOP

DELAY_60HZ:
    ; 735 семплов (1/60 секунды)
    LD DE,735
    LD (DELAY_COUNTER),DE
    JR PLAY_LOOP

DELAY_50HZ:
    ; 882 семпла (1/50 секунды)
    LD DE,882
    LD (DELAY_COUNTER),DE
    JR PLAY_LOOP
```

## Оптимизация для ZX Spectrum

### Преобразование частоты

VGM использует 44100 Гц, но на ZX Spectrum удобнее использовать прерывания 50 Гц.

Коэффициент пересчета:
```
Семплов на прерывание = 44100 / 50 = 882
```

### Упрощенная обработка задержек

Можно накапливать задержки и обрабатывать команды пока задержка не достигнет 882 семплов:

```asm
UPDATE_FRAME:
    LD DE,882             ; Семплов на фрейм
    LD (SAMPLES_LEFT),DE

PROCESS_COMMANDS:
    ; Обрабатывать команды
    CALL GET_NEXT_COMMAND
    
    ; Если задержка - вычесть из счетчика
    LD HL,(SAMPLES_LEFT)
    LD DE,(COMMAND_DELAY)
    OR A
    SBC HL,DE
    LD (SAMPLES_LEFT),HL
    RET C                 ; Фрейм завершен
    
    JR PROCESS_COMMANDS
```

## GD3 Теги (метаданные)

GD3 теги содержат информацию о треке:
- Название трека
- Имя игры
- Система
- Автор
- Дата выпуска
- и т.д.

Формат GD3 - Unicode строки с нулевым терминатором.

## Полезные константы

```asm
; VGM идентификатор
VGM_IDENT:  DB "Vgm "

; Версии VGM
VGM_V150:   EQU #00000150
VGM_V151:   EQU #00000151
VGM_V170:   EQU #00000170
VGM_V171:   EQU #00000171

; Команды
VGM_OPL3_P0:    EQU #5E  ; OPL3 порт 0
VGM_OPL3_P1:    EQU #5F  ; OPL3 порт 1
VGM_DELAY_WORD: EQU #61  ; 16-bit задержка
VGM_DELAY_60:   EQU #62  ; 735 семплов
VGM_DELAY_50:   EQU #63  ; 882 семпла
VGM_END:        EQU #66  ; Конец данных

; Частоты
SAMPLE_RATE:    EQU 44100
PAL_RATE:       EQU 50
NTSC_RATE:      EQU 60
SAMPLES_PAL:    EQU 882   ; 44100/50
SAMPLES_NTSC:   EQU 735   ; 44100/60

; Порты OPL3
OPL3_ADDR0:     EQU #C4
OPL3_DATA0:     EQU #C5
OPL3_ADDR1:     EQU #C6
OPL3_DATA1:     EQU #C7
```

## Примечания

1. **Сжатые файлы (.vgz)** - это VGM файлы сжатые gzip. Для воспроизведения их нужно сначала распаковать.

2. **Зацикливание** - многие VGM файлы содержат зацикливание. Проверьте Loop offset (#1C) в заголовке.

3. **Оптимизация** - для экономии памяти можно загружать VGM блоками с диска, а не целиком в память.

4. **Таймер** - точность воспроизведения зависит от стабильности системного таймера.

5. **OPL3 vs OPL2** - OPL3 (YMF262) имеет 2 банка регистров, OPL2 (YM3812) - только 1.
