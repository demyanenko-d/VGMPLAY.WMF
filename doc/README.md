## Быстрый старт

### Минимальная структура плагина

```asm
        ORG #6000
        ; Заголовок (512 байт)
        DS 16
        DB "WildCommanderMDL"
        DB #10
        ; ... остальные поля ...
        
        ORG #8000,#6200
PLUGIN:
        CALL GEDPL      ; Восстановить TXT режим
        ; Ваш код
        XOR A
        RET
```

### Компиляция

```batch
sjasm plugin.asm plugin.wmf
```

### Тестирование

1. Скопируйте .wmf файл в каталог WC на TSConfig
2. Нажмите F10 в WC для вызова меню плагинов
3. Выберите ваш плагин из списка

## Важные адреса и константы

```asm
; Системные адреса WC
WLD     EQU #6006       ; API точка вызова
ABT     EQU #6004       ; Флаг ESC
ENT     EQU #6005       ; Флаг ENTER
TMN     EQU #6009       ; Таймер

; Порты OPL3
OPL3_ADDR0  EQU #C4     ; Адрес регистра банк 0
OPL3_DATA0  EQU #C5     ; Данные банк 0
OPL3_ADDR1  EQU #C6     ; Адрес регистра банк 1
OPL3_DATA1  EQU #C7     ; Данные банк 1

; Основные функции API
GEDPL   EQU #0F         ; Восстановление TXT режима
PRWOW   EQU #01         ; Вывод окна
RRESB   EQU #02         ; Стирание окна
ESC     EQU #17         ; Проверка ESC
LOAD512 EQU #30         ; Загрузка файла
```

## Рекомендации по разработке

### Структура проекта

```
MyPlugin/
  ├── plugin.asm       # Основной код
  ├── header.asm       # Заголовок плагина
  ├── api.asm          # WC API макросы
  └── build.bat        # Скрипт компиляции
```

### Отладка

1. **Используйте окна для вывода отладочной информации**
   ```asm
   LD HL,DEBUG_MSG
   LD DE,#0101
   LD BC,10
   CALL PRSRW
   ```

2. **Проверяйте граничные условия**
   - Размер файла
   - Корректность заголовка
   - Доступность памяти

3. **Используйте эмулятор** Unreal Speccy Portable для первичной отладки

### Оптимизация

1. **Память**
   - Используйте страничную организацию для больших данных
   - Загружайте файлы блоками через LOAD512

2. **Скорость**
   - Используйте DMA для копирования больших блоков
   - Оптимизируйте критичные циклы

3. **Совместимость**
   - Проверяйте версию WC
   - Поддерживайте разные разрешения экрана (25/30/36 строк)

## Часто задаваемые вопросы

### Q: Как определить размер файла?
A: При входе в плагин размер в [DE,HL] (4 байта, little-endian).

### Q: Как работать с большими файлами?
A: Используйте потоковую загрузку LOAD512, загружайте блоками по мере необходимости.

### Q: Как организовать прерывания?
A: 
```asm
LD HL,INT_HANDLER
EXA
LD A,#FF
CALL WLD        ; INT_PL
EXA
```

### Q: Как сохранить состояние между запусками?
A: Выделите буферные страницы в заголовке (+34), они не очищаются между запусками.

### Q: Как работать с несколькими файлами?
A: Используйте систему потоков (STREAM, #39) для переключения между файлами.
