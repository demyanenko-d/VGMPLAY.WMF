# Wild Commander - Обзор Системы Плагинов

## Введение

Wild Commander (WC) - это приложение-навигатор с поддержкой плагинов для платформы TS-Config на базе ZX Spectrum. WC является основным приложением для работы с файловой системой на TSConfig.

## Версия системы плагинов

Текущая версия: **#10** (Plugin System version #10)

## Общая концепция

Плагины WC - это модульные расширения, которые добавляют функциональность для:
- Просмотра файлов различных форматов
- Воспроизведения музыки и видео
- Редактирования файлов
- Работы с ROM/Flash памятью
- Монтирования образов дисков
- И многое другое

## Типы запуска плагинов

При запуске плагина, в регистр **A** передается тип вызова:

| Тип | Описание |
|-----|----------|
| #00 | По расширению файла |
| #01 | При загрузке плагина (один раз) |
| #02 | По таймеру (скринсейвер) |
| #03 | Из меню запуска плагинов (F10) |
| #04 | Из меню выбора вьювера (SHIFT+F3/SHIFT+ENTER) |
| #12 | По нажатию F2 |
| #14 | По нажатию F4 |

## Входные параметры при запуске

При запуске плагина передаются следующие параметры:

```asm
; Входные параметры:
A       - тип вызова (см. таблицу выше)
[DE,HL] - длина файла в байтах
BC      - адрес строки с именем файла (для типов #00, #04, #14)
          формат: name(1-255), 0
IX      - адрес структуры активной панели

; Альтернативные регистры (только для #00,#04,#12,#14):
A'      - номер расширения (0-31) для типа #00
DE'     - количество файлов/каталогов в панели
HL'     - номер объекта из панели (от 1)
```

## Коды возврата из плагина

При выходе из плагина регистр **A** (или переменная `wc_exit_code` в SDCC) определяет действие:

| A | Действие |
|---|----------|
| 0 | Просто выход |
| 1 | Файл не опознан, передать другому плагину (только для типа #00) |
| 2 | Перейти на следующий файл (только для типа #00) |
| 3 | Перечитать каталог |
| 4 | Перейти на предыдущий файл (только для типа #00) |

В SDCC-плагине: `wc_exit_code = 0; return 0;` — WC читает значение A при выходе из `main()`.

## Вызов API функций

Все функции WC вызываются через точку доступа #6006:

```asm
WLD     EQU #6006

; Пример вызова функции PRWOW (вывод окна)
PRWOW   LD A,1
        JP WLD

; Пример вызова ESC (проверка нажатия ESC)
ESC     LD A,23
        JP WLD
```

## Работа со страницами памяти

### Резидент перехода (#6020)

Переключает страницу плагина и делает переход:

```asm
; Вход:
;   A  - номер страницы из выделенного блока (от 0)
;   HL - адрес перехода (#8000-#BFFF)

        LD A,1          ; Страница 1
        LD HL,#8000     ; Адрес
        CALL #6020
```

### Резидент вызова (#6028)

Вызывает код в другой странице плагина:

```asm
; Вход:
;   A  - номер страницы из выделенного блока (от 0)
;   HL - адрес вызова (#8000-#BFFF)
; Сохраняет все регистры кроме A и HL

        LD A,2          ; Страница 2
        LD HL,#A000     ; Адрес процедуры
        CALL #6028
```

## Важные замечания

1. **При входе в плагин может быть любой экран!** Необходимо вызывать `GEDPL` (#0F) / `wc_gedpl()` для переключения на текстовый режим.

2. **Вызывать GEDPL лучше после анализа файла**, когда убедились что файл подходит для плагина.

3. **Обязательно сохраняйте IX** при входе в плагин, если планируете использовать файловые функции.

4. **Win3 (#C000) сбрасывается после каждого вызова WC API** — внутри `CALL #6006` WC переключает окно `#C000` на текстовую страницу и **не восстанавливает** его после возврата. Если плагин переключил `#C000` через `mngcvpl()` / `mngc_pl()`, он обязан сделать это снова после любого UI-вызова (`gedpl`, `prwow`, `txtpr`, `rresb` и т.д.).

5. **GEDPL затирает #1000–#1447** в окне 0 — нельзя хранить там важные данные.
