# Модульная Архитектура VGM Player

## Структура проекта

Весь код разделен на независимые модули для лучшей поддерживаемости:

```
src/
├── vgmplayer.asm    - Главный файл (entry point + header)
├── wcapi.asm        - Обертки Wild Commander API
├── timing.asm       - Библиотека точного тактирования
├── opl3.asm         - Драйвер OPL3 (YMF262)
├── vgm.asm          - Парсер VGM формата
├── playback.asm     - Движок воспроизведения
└── ui.asm           - Пользовательский интерфейс
```

## Модули

### vgmplayer.asm - Главная точка входа
**Назначение**: Основной файл плагина, точка входа WC, инициализация
**Экспорт**: PLUGIN (entry point)
**Импорт**: Все остальные модули через INCLUDE

### wcapi.asm - Wild Commander API
**Назначение**: Обертки для всех функций WC API
**Функции**:
- `PRWOW` - Нарисовать окно
- `RRESB` - Удалить окно
- `PRSRW` - Вывести строку в окно
- `TXTPR` - Вывести форматированный текст
- `KBSCN` - Сканировать клавиатуру
- `ESC/SPKE/ENKE` - Проверка клавиш
- `LOAD512` - Загрузка блоков файла

### timing.asm - Точное тактирование
**Назначение**: Библиотека для микросекундных задержек
**Методы**:
1. **Line Interrupts** - прерывания по строкам (64 µs)
2. **CPU Cycle Counting** - подсчет тактов процессора
3. **Комбинированный** - линии для грубых задержек, такты для точных

**Возможности TSConfig**:
- 320 строк @ 15625 Hz = 64 µs/строка
- Порт #00AF - позиция прерывания (0-319)
- Порт #01AF bit 0 - включение line interrupt

**Функции**:
- `TIMING_INIT` - Инициализация системы тактирования
- `DELAY_US` - Задержка в микросекундах
- `DELAY_SAMPLES` - Задержка в VGM samples
- `DELAY_LINES` - Задержка на N строках видео
- `TIMING_CLEANUP` - Очистка

### opl3.asm - Драйвер OPL3
**Назначение**: Работа с YMF262 через порты #C4-#C7
**Функции**:
- `INIT_OPL3` - Инициализация чипа (включение OPL3 mode)
- `OPL3_DETECT` - Определение наличия OPL3
- `OPL3_RESET` - Сброс всех каналов
- `OPL3_WRITE_PORT0` - Запись в bank 0 (C4/C5)
- `OPL3_WRITE_PORT1` - Запись в bank 1 (C6/C7)
- `DELAY_OPL3_ADDR` - Задержка после записи адреса (16 циклов)
- `DELAY_OPL3_DATA` - Задержка после записи данных (41 цикл)

### vgm.asm - Парсер VGM
**Назначение**: Чтение и парсинг VGM файлов
**Функции**:
- `CHECK_FILE_SIZE` - Проверка размера (<3MB)
- `LOAD_VGM_FILE` - Загрузка файла в память
- `PARSE_VGM_HEADER` - Разбор заголовка VGM
- `PARSE_GD3_TAG` - Извлечение метаданных (UTF-16LE)

**Поддерживаемые поля**:
- Версия VGM (offset #08)
- OPL3 clock (offset #5C)
- Общее время samples (offset #18)
- Loop offset (offset #1C)
- Data offset (offset #34)
- GD3 tag (offset #14): game, track, author, system

### playback.asm - Движок воспроизведения
**Назначение**: Обработка VGM команд с точным тактированием
**Функции**:
- `PROCESS_VGM_FRAME` - Обработка команд на 1 кадр (882 samples)
- `GET_VGM_BYTE` - Чтение байта из VGM
- `RESTART_LOOP` - Рестарт с loop point
- `CLEANUP` - Очистка при выходе

**Поддерживаемые команды VGM**:
- `0x5E` - OPL3 port 0 write (99.01%)
- `0x5F` - OPL3 port 1 write (99.01%)
- `0x66` - END (99.01%)
- `0x61` - WAIT nn (16-bit) (98.02%)
- `0x62` - WAIT 735 samples (60Hz) (40.81%)
- `0x63` - WAIT 882 samples (50Hz) (42.09%)
- `0x70-0x7F` - WAIT 1-16 samples (23-35%)

### ui.asm - Пользовательский интерфейс
**Назначение**: Отображение окон и информации
**Функции**:
- `INIT_UI` - Инициализация интерфейса
- `REMOVE_UI` - Удаление окон
- `DISPLAY_VGM_INFO` - Показать метаданные
- `UPDATE_DISPLAY` - Обновить дисплей (forced)
- `UPDATE_DISPLAY_CONDITIONAL` - Обновить только при паузе >50ms
- `UPDATE_TIME_DISPLAY` - Обновить таймер
- `CHECK_KEYS` - Проверка клавиш
- `SHOW_ERROR` - Показать ошибку

**Стратегия обновления UI**:
- Обновление ТОЛЬКО во время больших пауз (>50ms = 2205 samples)
- Это предотвращает глитчи в музыке из-за CPU времени на UI
- Все изменения буферизируются и применяются за один раз

## Компиляция

### Главный файл собирает все модули
```asm
; vgmplayer.asm
        ORG #8000
        ; ... header ...
        
        INCLUDE "src/wcapi.asm"
        INCLUDE "src/timing.asm"
        INCLUDE "src/opl3.asm"
        INCLUDE "src/vgm.asm"
        INCLUDE "src/playback.asm"
        INCLUDE "src/ui.asm"
```

### Сборка
```bash
sjasm src/vgmplayer.asm vgmplay.wmf
```

## Зависимости

- **vgmplayer** → все модули
- **ui** → wcapi, timing
- **playback** → vgm, opl3, timing,wcapi
- **vgm** → wcapi
- **opl3** → timing (для delay)
- **timing** → независим
- **wcapi** → независим

## Глобальные переменные

Все глобальные переменные объявлены в главном файле `vgmplayer.asm`:
- Информация о файле (FILE_NAME, FILE_SIZE)
- VGM header данные (VGM_VERSION, VGM_CLOCK, etc.)
- Состояние воспроизведения (VGM_READ_PTR, SAMPLE_COUNTER)
- UI состояние (PAUSED, SONG_ENDED)

## Производительность

### CPU загрузка @ 7 MHz
- Обработка VGM команды: ~110 циклов
- Команд в кадре: 100-200 типично
- Overhead кадра: 11,000-22,000 циклов (7.8-15.7%)
- **Свободно**: 84-93% для тактирования/UI

### Память
- Код плагина: ~10 KB
- VGM файл: до 3 MB
- Рабочие буферы: ~2 KB
- **Итого**: ~3 MB максимум

## Оптимизации

1. **7 MHz режим процессора** (no-wait) - максимальная точность тактирования
2. **Line interrupts** - аппаратная поддержка точных задержек
3. **Условное обновление UI** - только при паузах >50ms
4. **Загрузка целиком** - весь VGM в память, без streaming
5. **Cycle-counted loops** - точные задержки OPL3

## Будущие улучшения

1. **32-bit адресация** - для файлов >64KB корректная поддержка
2. **VGZ декомпрессия** - поддержка gzip сжатых VGM
3. **Playlist** - очередь файлов
4. **Визуализатор** - отображение активности каналов
5. **Equalizer** - управление громкостью OPL3
