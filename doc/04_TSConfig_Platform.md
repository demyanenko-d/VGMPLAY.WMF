# TSConfig Platform Reference

## Обзор платформы

TS-Configuration - это расширенная конфигурация для ZX Spectrum с современными возможностями:

- Совместимость с Pentagon-128
- Расширенная графика (до 720x288, 256 цветов)
- Аппаратные спрайты и тайлы
- Ускорение Z80 (3.5/7/14 МГц)
- DMA контроллер
- До 4096 КБ RAM и 512 КБ ROM

## Память

### Страничная адресация

Процессор Z80 адресует 64 КБ памяти, разделенные на 4 окна по 16 КБ:

| Окно | Адреса CPU | Регистр страницы | R/W | Reset |
|------|------------|------------------|-----|-------|
| 0 | #0000..#3FFF | Page0 | W | #00 |
| 1 | #4000..#7FFF | Page1 | W | #05 |
| 2 | #8000..#BFFF | Page2 | R/W | #02 |
| 3 | #C000..#FFFF | Page3 | R/W | #00 |

**Доступно:**
- RAM: до 4096 КБ (256 страниц по 16 КБ)
- ROM: до 512 КБ (32 страницы по 16 КБ)

### Регистры управления памятью

Доступ к регистрам через порты **#nnAF**, где nn - номер регистра.

#### MemConfig (регистр #07)

```
Биты:
7:6 - LCK128[1:0] - режим порта #7FFD
5:4 - резерв
3   - W0_RAM - выбор ROM/RAM в окне 0
2   - !W0_MAP - режим маппинга окна 0
1   - W0_WE - разрешение записи в окно 0
0   - ROM128 - выбор страниц маппинга
```

#### Page0..3 (регистры #10-#13)

8-битные регистры, содержащие номер страницы для соответствующего окна.

## Порты ввода-вывода

### Схема адресации портов

TSConfig использует порты с адресом **#nnAF**, где:
- Младшие 8 бит (A0-A7) = #AF
- Старшие 8 бит (A8-A15) = номер регистра (nn)

### Базовые регистры TSConfig

> Источник: `src_sdcc/inc/tsconf.h` — `#define TSCONF_PORT(reg) (((uint16_t)(reg) << 8) | 0xAF)`

| Порт    | Константа                  | Режим | Название    | Описание                            |
|---------|----------------------------|-------|-------------|-------------------------------------|
| `#00AF` | `TSCONF_PORT_VCONFIG`      | W     | VConfig     | Видеорежим (VM, RRES и пр.)         |
| `#01AF` | `TSCONF_PORT_VPAGE`        | W     | VPage       | Страница видеобуфера                |
| `#06AF` | `TSCONF_PORT_TSCONFIG`     | W     | TSConfig    | FEQ, TMX и пр.                      |
| `#07AF` | `TSCONF_PORT_PALSEL`       | W     | PalSel      | Выбор палитры                       |
| `#0FAF` | `TSCONF_PORT_BORDER`       | W     | Border      | Цвет бордюра (биты 2:0)             |
| `#10AF` | `TSCONF_PORT_PAGE0`        | W     | **Page0**   | **Страница для окна #0000**         |
| `#11AF` | `TSCONF_PORT_PAGE1`        | W     | **Page1**   | **Страница для окна #4000**         |
| `#12AF` | `TSCONF_PORT_PAGE2`        | RW    | **Page2**   | **Страница для окна #8000**         |
| `#13AF` | `TSCONF_PORT_PAGE3`        | RW    | **Page3**   | **Страница для окна #C000**         |
| `#20AF` | `TSCONF_PORT_SYSCONFIG`    | W     | **SysConfig** | **CPU частота + кэш глобально**   |
| `#21AF` | `TSCONF_PORT_MEMCONFIG`    | W     | MemConfig   | Конфигурация памяти / совместимость |
| `#2AAF` | `TSCONF_PORT_INTMASK`      | W     | INTMask     | Маска прерываний (FRAME/LINE/DMA)   |
| `#2BAF` | `TSCONF_PORT_CACHECONFIG`  | W     | CacheConfig | Кэш по окнам                        |

> ⚠️ **Частая ошибка:** Page0–Page3 = `#10AF`–`#13AF`, SysConfig = `#20AF`. Адреса `#00AF`/`#01AF`/`#0FAF` для страниц — неверны!

### Порты OPL3 — YMF262

**OPL3 чип располагается по портам #C4–#C7:**

| Порт  | Назначение                     |
|-------|--------------------------------|
| `#C4` | Bank 0: регистр адреса         |
| `#C5` | Bank 0: данные                 |
| `#C6` | Bank 1: регистр адреса         |
| `#C7` | Bank 1: данные                 |

**Регистр NEW (Bank 1, reg 0x05):**
- `NEW=1` (0x01) → OPL3 mode: биты L/R в регистрах `C0–C8` управляют маршрутизацией звука
- `NEW=0` (0x00) → OPL2-compat: L+R всегда активны (обязательно для файлов OPL2/OPL1!)

```c
// inc/opl3.h
void opl3_write_b0(uint8_t reg, uint8_t data); // Bank 0 (#C4/#C5)
void opl3_write_b1(uint8_t reg, uint8_t data); // Bank 1 (#C6/#C7)
void opl3_init(void);  // сброс + NEW=1

// Для OPL2/OPL1 VGM: после opl3_init() нужно:
opl3_write_b1(0x05, 0x00);  // NEW=0 — иначе тишина!
```

### Порты AY-3-8910 / YM2149

| Порт    | Назначение                               |
|---------|------------------------------------------|
| `#FFFD` | Регистр адреса (`OUT (#FFFD), A`)        |
| `#BFFD` | Запись данных (`OUT (#BFFD), A`)         |
| `#FFFD` | Чтение значения регистра (`IN A,(#FFFD)`) |

TurboSound (два AY): второй чип выбирается через порт `#FF` (биты 1:0).

### Порты SAA1099 — MultiSound card

| Чип  | Порт адреса | Порт данных |
|------|-------------|-------------|
| SAA1 | `#00FF`     | `#01FF`     |
| SAA2 | `#02FF`     | `#03FF`     |

**Пример записи в регистр OPL3:**
```asm
; Запись в регистр банка 0
    LD BC,#C4      ; Порт адреса
    LD A,reg       ; Номер регистра
    OUT (C),A
    LD B,#C5       ; Порт данных
    LD A,value     ; Значение
    OUT (C),A

; Запись в регистр банка 1
    LD BC,#C6      ; Порт адреса банка 1
    LD A,reg       ; Номер регистра
    OUT (C),A
    LD B,#C7       ; Порт данных банка 1
    LD A,value     ; Значение
    OUT (C),A
```

## Процессор

### Частоты CPU

Доступны 3 режима работы:

| Режим | Частота |
|-------|---------|
| 00 | 3.5 МГц |
| 01 | 7.0 МГц |
| 10 | 14.0 МГц |

Управление через регистр **SysConfig #20AF** (биты 1:0):

```c
// tsconf.h
#define TSCONF_CPU_3_5MHZ   0x00
#define TSCONF_CPU_7MHZ     0x01
#define TSCONF_CPU_14MHZ    0x02
#define TSCONF_CACHE_EN     0x04   // бит 2 — глобальный кэш
```

### Кэш

512 байт кэш-памяти для ускорения работы на 14 МГц.

Управление:
- Глобально: бит 2 регистра **SysConfig**
- По окнам: регистр **CacheConfig**

**Важно:** После DMA операций необходима инвалидация кэша!

```asm
; Инвалидация кэша
    LD HL,#FE00
    LD DE,#FE00
    LD BC,#0200
    LDIR
```

## Прерывания

### Источники прерываний

TSConfig поддерживает несколько источников INT:

| Источник | Приоритет | Адрес (IM2) | Описание |
|----------|-----------|-------------|----------|
| FRAME | 0 | #FF | Кадровое прерывание |
| LINE | 1 | #FD | Строчное прерывание |
| DMA | 2 | #FB | Завершение DMA |

### Регистр маски прерываний (INTMask)

```
Биты:
2 - DMA interrupt enable
1 - LINE interrupt enable
0 - FRAME interrupt enable

Reset: #01 (только FRAME включен)
```

### ISR плагина — собственный таймер

Плагин **не использует** кадровое прерывание TSConfig. Вместо него устанавливается собственный ISR через `wc_int_pl(mode)` с частотой:

| Параметр           | Значение                          |
|--------------------|-----------------------------------|
| Частота ISR        | **1367 Гц**                       |
| T-states/тик       | **2560 T** @ 3.5 МГц              |
| Тиков на TV-кадр   | **28** (28 × 2560 = 71680 T)      |
| Связь с VGM        | 44100 / 32 = 1378 ≈ 1367 Гц       |

```c
// inc/isr.h
#define ISR_FREQ             1367  // тиков/сек = 3500000 / 2560
#define ISR_TICKS_PER_FRAME  28
#define VGM_SAMPLE_SHIFT     5    // 44100 >> 5 = 1378 ≈ ISR_FREQ
```

### Частота прерываний TSConfig (FRAME)

- FRAME: зависит от видео режима
  - 50 Гц (312 строк PAL)
  - 60 Гц (262 строки NTSC)
  - ~48.8 Гц (320 строк Pentagon)

## DMA Контроллер

TSConfig имеет мощный DMA контроллер для быстрого копирования данных:

- RAM → RAM
- Device → RAM
- RAM → Device
- RAM → CRAM (палитра)
- RAM → Sprite File

### Основные регистры DMA

| Регистр | Описание |
|---------|----------|
| DMA_SAL | Source Address Low |
| DMA_SAH | Source Address High |
| DMA_SAX | Source Address eXtended (страница) |
| DMA_DAL | Destination Address Low |
| DMA_DAH | Destination Address High |
| DMA_DAX | Destination Address eXtended (страница) |
| DMA_T | Transaction count (количество бёрстов) |
| DMA_N | Burst size (размер бёрста) |

## Видео система

### Видео режимы

| Режим | Разрешение | Цвета |
|-------|------------|-------|
| ZX | 256x192 | 16 (атрибуты) |
| 16c | 256x192, 320x200, 320x240, 360x288 | 16 |
| 256c | то же | 256 |
| TXT | 80x25, 80x30, 80x36 | 256 |

### Палитра

256 ячеек палитры, формат RGB555 (15 бит цвета).

### Спрайты

- До 85 спрайтов на строку
- Размеры: 8x8 до 64x64 пикселей
- До 3 плоскостей спрайтов
- До 16 палитр на строку

## Совместимость с Pentagon-128

### Порт #7FFD

Поддерживается для совместимости:

```
Биты:
7:6 - дополнительные биты страницы (512K/1024K)
5   - LOCK (блокировка переключения)
4   - ROM128 (выбор ROM)
3   - SCR (экран 5 или 7)
2:0 - номер страницы для #C000
```

### Режимы совместимости

| LCK128 | Режим | Описание |
|--------|-------|----------|
| 00 | 512 КБ | Pentagon-512 |
| 01 | 128 КБ | Pentagon-128 |
| 10 | Авто | Определяется автоматически |
| 11 | 1024 КБ | Расширенный режим |

## Пример инициализации для плагина

```asm
; Базовая инициализация для музыкального плагина

INIT_PLUGIN:
    ; Сохранить текущее состояние
    LD A,(#6002)      ; Страница в окне 2
    PUSH AF
    LD A,(#6003)      ; Страница в окне 3
    PUSH AF

    ; Восстановить TXT режим
    LD A,#0F
    CALL WLD          ; GEDPL
    
    ; Установить частоту CPU
    LD B,#00          ; Z80 frequency
    LD C,%01          ; 7 MHz
    LD A,#0E
    CALL WLD          ; TURBOPL
    
    ; Настроить прерывания
    LD HL,INT_HANDLER
    EXA
    LD A,#FF
    CALL WLD          ; INT_PL
    EXA
    
    ; Инициализация OPL3
    CALL INIT_OPL3
    
    RET

; Обработчик прерываний
INT_HANDLER:
    PUSH AF, BC, DE, HL
    ; Ваш код обработки музыки
    CALL UPDATE_MUSIC
    POP HL, DE, BC, AF
    EI
    RETI

; Инициализация OPL3
INIT_OPL3:
    ; Сброс OPL3
    LD BC,#C4
    XOR A
    OUT (C),A
    LD B,#C5
    OUT (C),A
    ; Дополнительная инициализация...
    RET
```
